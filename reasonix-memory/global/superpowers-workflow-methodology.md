**来源：** [meisijiya/Efficient-Reasonix](https://github.com/meisijiya/Efficient-Reasonix) — 整合自 [obra/superpowers](https://github.com/obra/superpowers) (208k⭐)

**7 步标准工作流：**
1. brainstorming — 澄清需求，2-3 方案，分段展示设计
2. writing-plans — 拆成 2-5 分钟可执行小任务
3. subagent-driven-development — 派分子代理 + 双审
4. test-driven-development — RED→GREEN→REFACTOR
5. requesting-code-review — 对照计划审查
6. verification-before-completion — 确保真修好了
7. finishing-a-development-branch — 合并/PR 决策

**4 条哲学：**
- 测试驱动开发（永远先写测试）
- 系统化胜于随意（流程优于碰运气）
- 降低复杂度（简单是首要目标）
- 证据胜于声称（验证之后才能说完成）

**Why:** 这套方法论已在 208k⭐ 的项目中验证，能显著提升 AI 编码 agent 的输出质量
