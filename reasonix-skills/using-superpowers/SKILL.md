# Superpowers 标准工作流

## 核心流程

严格遵循以下 7 步，每一步完成后自动进入下一步：

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1. 理解需求 | brainstorming | 澄清需求边界和验收标准 |
| 2. 探索仓库 | explore | 定位相关文件，理解架构 |
| 3. 拆分任务 | brainstorming | 拆分为独立小任务单元 |
| 4. 编写计划 | writing-plans | 按依赖顺序列出实现步骤 |
| 5. 执行实现 | subagent + TDD | 逐个分派，一次一个 |
| 6. 验证调试 | verify + debug | 测试验证，两轮审查 |
| 7. 提交准备 | finish-branch | 整理提交信息 |

## 强制规则

- **在任何响应之前**，检查是否有技能适用
- 即使 1% 的可能性也要调用技能检查
- 设计批准 ≠ 可以开始实现（仍需文档→自检→审查→writing-plans）

## 红线

- "这只是一个简单的问题" → 检查技能
- "设计已批准，直接实现" → 仍需写文档、自检、用户审查
