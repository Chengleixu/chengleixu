# 头脑风暴

## 流程

1. **探索上下文** — 检查文件、文档、最近的 commit
2. **提出澄清问题** — 每次一个，了解目的/约束/成功标准
3. **提出 2-3 种方案** — 附带权衡分析和推荐
4. **展示设计** — 分节展示，每节获批准
5. **编写设计文档** — 保存到 `docs/superpowers/specs/` 并 commit
6. **规格自检** — 检查占位符/矛盾/模糊性
7. **用户审查** — 请用户审查规格文件
8. **过渡** — 调用 writing-plans

## 核心原则

- 每次一个问题，优先选择题
- 严格 YAGNI，移除不必要功能
- 增量验证，展示设计→批准→继续
- 小改动也要走流程
